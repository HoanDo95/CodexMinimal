# Legacy Compatibility

IDSD is the default workflow.

Use legacy SDD only when:

- the user explicitly requests SDD
- an existing team contract requires a traditional spec
- a profile-specific planner is already part of an approved plan

When legacy SDD is used, treat it as an evidence artifact inside IDSD, not as the controlling workflow.

Use TDD when behavior needs executable evidence, but do not let tests replace intent, risk, or decision capture.
