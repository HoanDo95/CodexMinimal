# Naming Conventions

Suggested NestJS names:

- `<feature>.module.ts`
- `<feature>.controller.ts`
- `<feature>.service.ts`
- `dto/<action>-<resource>.dto.ts`
- `entities/<resource>.entity.ts`
- `repositories/<resource>.repository.ts`
- `test/unit/<feature>/<feature>.service.spec.ts`

Prefer explicit names over generic names.
