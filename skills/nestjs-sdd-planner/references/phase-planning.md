# Planning Handoff Notes

The spec stage should not try to replace the implementation plan.

At spec time, record only the minimum handoff notes that help the next planning step:

- risky areas that deserve isolated phases later
- protected-file or integration concerns
- sequencing constraints that the planner must not ignore
- verification expectations that should shape the plan

Do not expand these notes into a full phase plan here.
That belongs to the later `repo-phase-orchestrator` stage.
